# Bad User Story
Create a login functionality.
## No value 
## No user
## No testability
## Open to interpretation (Forces developers to guess)

# Good User Story
As a registered user, I want to log in to the system, so that I can access my personal dashboard
# Acceptance Crieria
- Users can enter email and password 
- If credentials are invalid, an error message is shown
- Successful login redirects to dashboard

# Comparing Vague vs Testable User Stories
## Vague user story
The system must be user friendly
### Subjective
### Not Testable
### High potential for conflict

## Testable User Story
As a user, I want to be able to clear error messages, so that I know how to fix input mistakes
